🔐 Cryptography & Security

SHA3-256 Hashing Algorithm

  • ✅ 256-bit security — resistant to known attacks
  • ✅ Collision resistance — impossible to find same hash
  • ✅ Avalanche effect — small change = completely different hash
  • ✅ NIST standard — approved by National Institute of Standards
  • ✅ Different from Bitcoin's SHA-256 — more modern design

ECDSA Signatures (secp256k1)

  • Private key generation and secure local storage
  • Public key derivation for DX... addresses
  • Transaction signing and on-chain verification
  • Non-repudiation guarantees

🌐 Network Architecture

Fully Decentralized — No Central Authority

  • 🔗 Peer-to-Peer Network — direct node communication
  • 🗳️ Consensus by Mining — distributed block validation
  • 📊 Distributed Ledger — every node holds full chain copy
  • 🔒 Trustless System — no central entity needed
  • 🛡️ Censorship Resistant — no one can block transactions

What Decentralized Means for Users

  • ✅ No registration or KYC required
  • ✅ Anyone can run a full node
  • ✅ Open source and auditable by anyone
